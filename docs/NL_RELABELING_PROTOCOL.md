# 자연어 추천 평가 — 독립 재라벨링 프로토콜

## 왜 필요한가 (순환논증 문제)

기존 평가셋(`eval_tfidf_scenarios.json`)은 **MOOD_MAP을 만든 사람이 정답(`expected_relevant`)도 같은 토큰으로 라벨**했다. 그래서 "자연어 기능을 켜니 점수가 올랐다"가 **자연어를 이해해서가 아니라 동어반복(자기충족)**일 수 있다. 교수가 "정답도 네가 만들었잖아"라고 반박하면 방어가 안 된다.

→ **해결: 룰(MOOD_MAP/expand_context)을 보지 않은 평가자가 정답을 독립적으로 다시 라벨**하고, 평가자 간 일치도(Cohen's κ)를 보고한다.

## 절차

1. **대상 셋**: `backend/tests/fixtures/nl_relabel_set.json` 의 30개 쿼리 (패러프레이즈 15쌍 포함).
2. **평가자**: 2~3명. **조건 — `context_expander.py`(MOOD_MAP)와 `eval_tfidf_scenarios.json`을 절대 보지 않는다.** 추천 알고리즘도 모르는 게 이상적.
3. **자료 제공**: 평가자에게 (a) 자연어 쿼리 문장, (b) 1667개 레시피 이름 목록(또는 검색 UI)만 준다. 코드·룰·기존 정답은 차단.
4. **라벨링 기준**: 각 쿼리에 대해 "이 자연어 의도에 **적절한** 레시피"를 레시피 목록에서 고른다. 3점 척도 권장:
   - `2` = 매우 적절 (정확히 그 의도)
   - `1` = 어느 정도 적절
   - `0` = 부적절
5. **집계**:
   - 평가자별 라벨을 모아 **다수결(2명 이상이 1점 이상)**을 `expected_relevant` 로 채택.
   - **Cohen's κ** (2인) 또는 **Fleiss' κ** (3인 이상) 로 일치도 산출. κ ≥ 0.6 이면 "상당한 일치"로 보고.
6. **반영**: 합의된 정답으로 `nl_relabel_set.json` 의 `expected_relevant` 를 갱신(또는 `expected_relabeled` 필드 추가)한 뒤 `evaluate_semantic_nl.py` 재실행.

## 보고서에 쓸 문장 (예)

> "자연어 30개 쿼리의 적절 레시피는 추천 알고리즘과 MOOD_MAP을 보지 않은 평가자 3명이 독립 라벨링했으며, Fleiss' κ = 0.XX(상당한 일치)였다. 이 합의 정답을 기준으로 TF-IDF와 의미 임베딩의 nDCG@5를 paired t-test로 비교했다."

## 한계 명시 (정직)

- 표본 n=30 (패러프레이즈 15) — 소규모. effect size(Cohen's d)·신뢰구간을 함께 보고하고 "탐색적 결과"로 표현.
- 평가자가 학부생/동료라 도메인 전문가가 아님 → 외적 타당도 제한. 가능하면 소규모 사용자 설문(실사용자 N명이 추천 만족도 5점 척도) 보강.
