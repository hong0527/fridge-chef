# 자연어 추천 — 의미 임베딩 도입 & 측정 결과 (실측)

> 재현: `cd backend && python scripts/rebuild_corpus_cache.py && python scripts/precompute_embeddings.py && python scripts/evaluate_semantic_nl.py`
> 설정: 코퍼스 1667, NL retrieval_k=20, nl_weight=0.35, Top-5, 쿼리 30개(패러프레이즈 15), Gemini 폴백 강제.

## 0. 한눈에

- **진짜 병목은 "어떤 임베딩이냐"가 아니라 "후보 생성(retrieval)"이었다.** 자연어 정답의 다수가 재료 overlap·theme 하드필터에서 **랭킹 전에 탈락**했다.
- **해결: 자연어 의미검색을 후보 생성에 주입(NL retrieval).** 정답 미도달이 **20/30 → 4/30** 으로 급감, 전체 nDCG@5 가 약 **2.5배(0.124→0.310)** 상승.
- TF-IDF·의미(e5)·하이브리드 중 **이 평가셋에선 통계적 유의차 없음(n=30)**. 단 **어휘 겹침 없는 진짜 패러프레이즈는 의미/하이브리드만 잡고 TF-IDF는 0점**.

## 1. retrieval 주입 효과 (구조 병목 해소)

| 지표 | 주입 OFF (재정렬 전용) | 주입 ON (k=20) |
| --- | ---: | ---: |
| 정답이 후보(Top-10) 미도달 | **20/30** | **4/30** |

## 3. 핵심 — 진짜 패러프레이즈에서의 차이 (어휘 겹침 0)

| 쿼리 | TF-IDF | e5 | Hybrid |
| --- | ---: | ---: | ---: |
| Q18 "기념일에 근사하게 구운 고기 요리" | **0.000** | **0.509** | 0.360 |
| Q10 "학창시절 떡볶이 같은 얼큰달콤한 거" | **0.000** | **0.345** | 0.214 |
| Q14 "새콤한 토마토 소스 면 요리" | **1.000** | 0.000 | **1.000** |
| Q26 "바삭하게 튀긴 새콤달콤 중국요리" | **0.869** | 0.000 | 0.339 |

→ **쿼리에 음식 단어가 그대로 있으면 TF-IDF, 의미만 통하는 표현이면 e5가 이긴다. Hybrid 는 양쪽을 모두 회복** (Q14=1.0 유지, Q18=0.36 회복). Hybrid 가 가장 견고(치명적 0점 최소).

## 4. 정직한 한계

1. **자체 정답지 편향**: `expected_relevant` 를 레시피 이름 부분문자열로 작성 → 쿼리가 음식 단어를 포함하면 TF-IDF에 유리하게 기운다. 전체 평균에서 TF-IDF가 높은 주 원인. → **독립 블라인드 재라벨링 필수**(`NL_RELABELING_PROTOCOL.md`). 중립 정답지면 의미/하이브리드가 유리해질 가능성.
2. **표본 n=30** → 통계적 유의차 도출엔 부족. effect size·CI·"탐색적" 표현 병기.
3. 잔여 미도달 4건(Q05/Q06/Q21/Q22)은 맵기·난이도·조리시간 필터 또는 코퍼스 이름 불일치 → retrieval_k 확대/필터 완화로 추가 개선 여지.

## 5. 권장 설정 & 운영 주의

- **권장(평가·시연)**: `EMBEDDING_BACKEND=hybrid`, `NL_RETRIEVAL_K=20`, `NL_WEIGHT=0.35`. 하이브리드가 어휘+의미 모두 커버해 가장 견고.
- **운영(Render 512MB)**: e5 모델 상주 메모리 부담 → 기본 `tfidf` 유지. 의미/하이브리드는 메모리 여유 환경 또는 사전계산 캐시 + int8/ONNX 경량화 후 적용.
- 안전: 알레르기 0%·국가 선호 hard filter 는 retrieval 주입에서도 유지(회귀 테스트 통과 402건).
