# FridgeChef 시연 가이드 (팀원용 / 영상 촬영용)

> 목적: 시연 영상 촬영 + 라이브 시연 시 **무엇을 입력하면 무엇이 나오는지**, 필터가 잘 도는지, 중간에 오류를 어떻게 잡는지 정리.
> 기준: main 배포본 (nl_weight=0.30, nl_retrieval_k=12, gemini-2.5-flash, TF-IDF).

---

## 0. 시연 전 체크 (필수)

| 항목 | 확인 |
|---|---|
| **운영 서버 살아있나** | `https://fridgechef-backend-d2ae.onrender.com/health` → `{"status":"ok","recipe_count":1667}` |
| **Gemini 키** | Render Environment의 `GEMINI_API_KEY` = 작동하는(크레딧 있는) 키. 없으면 자연어가 약하게만 작동(폴백) |
| **Render 콜드스타트** | 무료티어는 일정시간 미사용 시 잠듦 → 첫 요청 5~8초 느림. **시연 1~2분 전 health 한 번 호출해 깨워두기** |

---

## 1. 시작 냉장고 재료 (복붙용)

여러 추천이 잘 나오도록 **흔하고 겹침 많은 재료 15개**로 시작 권장:

```
김치, 돼지고기, 두부, 대파, 양파, 마늘, 계란, 밥, 고추장, 떡, 어묵, 콩나물, 당근, 감자, 소고기
```

- 이 정도면 자연어를 바꿔도 빈 결과 없이 다양한 추천이 나옴.
- 재료가 너무 적으면(3~4개) 추천이 적거나 빌 수 있음 → 시연엔 10개 이상 권장.

---

## 2. 시연 시나리오 (같은 재료, 자연어만 바꿔 보여주기) ⭐

**핵심 메시지: "냉장고는 그대로인데 자연어 요구에 따라 추천이 달라진다"**

| # | 자연어 입력 | 기대 추천(예시) | 보여줄 포인트 |
|---|---|---|---|
| 1 | `비 오고 쌀쌀한데 따뜻한 국물 요리` | 김치수제비·된장채소수제비·콩나물국밥 | 날씨/상황어 → **국물류** |
| 2 | `오늘 스트레스 받아서 맵고 얼큰한 거` | 참치김치찌개·굴비매운탕·오징어삼겹살볶음 | 감정 → **매운 음식** (단어 없어도) |
| 3 | `퇴근하고 피곤해서 빠르고 간단한 야식` | 라볶이·김치동그랑땡 | 상황 → **간단/야식** |
| 4 | `주말에 가족이랑 특별하게 푸짐한 요리` | (B쪽) 돼지갈비구이·닭고기콩나물덮밥 | 의도 → **특별한 요리** |
| 5 | `다이어트 중 가볍고 담백한데 든든한` | 두부전·콩나물국밥·(B)두부구이와버섯볶음 | 건강 의도 (※아래 한계 참고) |

> 💡 촬영 팁: **2번(감정→매운)**과 **1번(날씨→국물)**이 가장 극적. "오늘 짜증나"처럼 음식 단어 0개 입력이 매운 음식으로 연결되는 게 임팩트 큼.

---

## 3. 필터 시연 (검증 완료 — 잘 돔)

같은 재료/자연어에서 **선호(필터)만 바꿔** 결과가 바뀌는 것 보여주기:

| 필터 | 시연 방법 | 검증 결과 |
|---|---|---|
| **매운맛** | 맵기 1 ↔ 5 | ✅ 1→순한 음식, 5→매운 음식 |
| **조리시간** | 최대 10분 ↔ 60분 | ✅ 10분→빠른 요리만, 60분→다양 |
| **알레르기** | 계란/땅콩 등 설정 | ✅ **해당 재료 포함 레시피 0건** (안전 보장) |
| **국가** | 한식 ↔ 양식 ↔ 중식 | ✅ 선택 국가만 나옴 |
| **난이도** | 초보 ↔ 고급 | ✅ 난이도 맞는 것만 |

> ⚠️ **다이어트(저칼로리) 필터는 약함** — ON/OFF 차이가 잘 안 보임(가중치가 낮음). 시연에서 다이어트 ON/OFF 비교는 **피하는 게 안전**. (개선 예정)

---

## 4. 시연 중 오류 체크리스트 🔎

문제 발생 시 빠르게 원인 파악:

| 증상 | 원인 | 대응 |
|---|---|---|
| **추천이 비어 나옴** | 재료 너무 적음 / 필터 너무 빡빡(맵기·국가·조리시간 조합) | 재료 늘리기, 필터 완화 |
| **첫 요청이 매우 느림(8초+)** | Render 콜드스타트 | 시연 전 미리 깨워두기 (health 호출) |
| **추천 이유가 "보유한 ~ 추가 시 완성" 템플릿만** | Gemini 이유 호출이 느려 폴백 | 정상 동작(레시피는 정확). 재시도 시 진짜 이유 나옴 |
| **자연어 바꿔도 추천 안 변함** | Render에 작동 키 없음(폴백) / 입력이 너무 짧음 | 키 확인, 자연어 구체적으로 |
| **"다이어트"인데 안 가벼운 거** | diet 필터 약함(알려진 한계) | 시연에서 회피 |
| **같은 재료에 라볶이가 자주 1등** | A(냉털)는 재료겹침 비중이 큼(라볶이 독식) | 알려진 약점, 개선 예정 |

---

## 5. 신뢰성 (재현성)

- **같은 입력 → 같은 추천** (TF-IDF + 고정 가중치 + temperature 0). 실측으로 2회 동일 확인.
- 단 **이유 글**은 Gemini 응답 타이밍에 따라 진짜 문장/템플릿이 갈릴 수 있음(레시피는 항상 동일).

---

## 6. 동작 원리 한 줄 (발표용)

> 자유 자연어 → **Gemini가 음식 의도로 번역**("오늘 짜증나"→"맵고 얼큰한 음식") → **TF-IDF로 1667 레시피와 매칭** → 알레르기·국가·맵기·난이도·조리시간 **하드필터** + (재료 70% + 자연어 30%) **가중 랭킹** → Gemini가 추천 이유 작성.
> (의미임베딩 e5는 512MB 메모리상 운영 불가 → Gemini 번역 + TF-IDF로 대체)

---

## 7. 로컬에서 시연/테스트 재현 (개발자용)

```bash
cd backend
# 1. 코퍼스 캐시 + (필요시) 임베딩
python scripts/rebuild_corpus_cache.py
# 2. 종합 평가 보고서
python scripts/full_eval_1667.py        # nDCG@5/Recall@5/MRR + 성능 + 알레르기 0%
# 3. 회귀 테스트
pytest -q                                # 409 통과 기준
```

> 평가 한계(정직): 자연어 정답지는 자체 라벨이라 편향 가능, LLM-judge는 Gemini 프록시. **객관 신뢰성은 독립 인간 재라벨링(`NL_RELABELING_PROTOCOL.md`)으로 보강 예정.**
